# Knowledge

Durable, reusable knowledge that doesn't fit cleanly under product,
architecture, or operations. Two kinds:

- **[Learnings](./learnings/new-things.md)** — techniques that are novel in
  this project, each with a code-grounded gotcha and an authoritative
  external source. Lean on the source for the concept; keep the gotcha here.
- **[Failed approaches](./failed-approaches/node-24-lighthouse-12.md)** —
  things that were tried or considered and didn't work, with the reason.
  Kept so the same path isn't re-explored.

## Rules

- Don't re-explain concepts that have a definitive external source — link
  and move on.
- Each entry should answer: *what*, *why it matters to THIS project*, and
  *where in this codebase*.
- Failed approaches must record **why** they failed, not just that they did.
