# Evidence Research

Use this workflow whenever asked to research, compare tools, recommend a
product, investigate a technical claim, or summarize an external topic.

1. Restate the decision to be made, success criteria, constraints, and deadline.
2. Form a short research plan with the 3-6 questions that would change the
   decision.
3. Gather primary sources first. Verify volatile facts with current official
   sources. Cross-check consequential claims with an independent source.
4. Build a claim ledger while researching: claim, evidence URL, source type,
   date, confidence, and caveat.
5. Distinguish facts from inferences. State what evidence would reverse a
   recommendation.
6. Write a report to `reports/YYYY-MM-DD-<topic>.md` with these sections:
   `Decision summary`, `Scope`, `Findings`, `Options`, `Risks`, `Recommendation`,
   `Open questions`, and `Sources`.
7. Keep source links adjacent to claims. Do not pad a report with generic advice
   or unsupported rankings.

For technical products, include deployment model, credential model, integration
surface, cost or quota model when public, security implications, maintenance
burden, and a reversible adoption path.
