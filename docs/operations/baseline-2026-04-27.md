## How to re-run on day 7 (2026-05-04)

```bash
# Auth: needs $CLOUDFLARE_API_TOKEN with Account.Workers Scripts:Read,
#       Account.D1:Read, Account.R2:Read, Account.KV:Read, Account.Analytics:Read.
ACCT=7d048325699a5acddb44d3be31cf6ba9
START=$(date -u -v-7d +%Y-%m-%dT%H:%M:%SZ)   # macOS; Linux: -d '7 days ago'
END=$(date -u +%Y-%m-%dT%H:%M:%SZ)

# Workers
cat > /tmp/q.json <<EOF
{"query":"query(\$a:String!,\$s:Time!,\$e:Time!){viewer{accounts(filter:{accountTag:\$a}){workersInvocationsAdaptive(filter:{datetime_geq:\$s,datetime_leq:\$e},limit:10000){sum{requests subrequests errors}quantiles{cpuTimeP50 cpuTimeP99}dimensions{scriptName status}}}}}","variables":{"a":"$ACCT","s":"$START","e":"$END"}}
EOF
curl -sS -X POST -H "Authorization: Bearer $CLOUDFLARE_API_TOKEN" \
  -H "Content-Type: application/json" \
  https://api.cloudflare.com/client/v4/graphql --data @/tmp/q.json | jq

# D1 (replace `workersInvocationsAdaptive{...}` with `d1AnalyticsAdaptiveGroups(filter:{datetime_geq:$s,datetime_leq:$e},limit:10000){sum{readQueries writeQueries rowsRead rowsWritten}dimensions{databaseId}}`)
# R2: r2OperationsAdaptiveGroups + r2StorageAdaptiveGroups
# KV: kvOperationsAdaptiveGroups
# Pages: pagesFunctionsInvocationsAdaptiveGroups
```

Compare day-7 totals against this baseline. **Decision rule:**

- Keep Paid if: any worker exceeded 100k/day on any one day (Free cap), OR Durable Objects went into production use, OR any single project added Containers/Workflows/Vectorize/Browser Rendering, OR D1 exceeded 25M rows-read/month projection, OR you needed `wrangler tail` for prod debugging.
- Cancel Paid if: aggregate worker requests stay <500k/week, no DO/Container/Workflow code shipped, no R2 GB-class storage, no Vectorize index in use.
